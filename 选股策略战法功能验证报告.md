## ✅ 实际存在的「策略」系统（非股票）

代码库中的「策略」一词全部指向 **内容推荐系统的排序策略**，与股票无关：

### 推荐排序融合策略 (`src/recommendation/ranking.py`)

```
┌─────────────┐   权重   ────────────────────────────
│ ItemCF 协同过滤│ ──35%──→ │ 用户行为相似度召回        │
│ UserCF 协同过滤│ ──15%──→ │ 相似用户偏好召回          │
│ Content 内容匹配│ ──25%──→ │ 标签/分类内容相似度       │
│ Trending 热度 │ ──20%──→ │ EMA 平滑热度排行榜       │
│ Random 探索   │ ──10%──→ │ Boltzmann 分布探索流量    │
└─────────────┘         ────────────────────────────
         ↓
   MMR 多样性重排 (λ=0.7)
         ↓
   业务规则过滤 → 最终推荐列表
```

| 参数 | 值 | 说明 |
|------|-----|------|
| 多样性 lambda | 0.7 | 相关性 vs 多样性平衡 |
| 新内容加成 | 1.5x | 发布3天内内容加权 |
| 最大作者占比 | 30% | 同一作者去重 |
| 最大分类占比 | 50% | 分类打散 |
| Softmax 温度 | 0.3 | 探索-利用分布 |

### 文件存储策略 (`src/storage/manager.py`)

| 策略名 | 选择逻辑 |
|--------|---------|
| DEFAULT | 默认提供商优先 |
| PERFORMANCE | 延迟最低优先 |
| COST | 成本最低优先 |
| DURABILITY | 可靠性最高优先 |
| GEO | 区域就近优先 |

### 缓存策略 (`src/core/cache.py`)

- L1 内存缓存 (TTLCache, maxsize=1000, TTL=300s)
- L2 Redis 分布式缓存
- pickle 序列化 / 反序列化
- 布隆过滤器防穿透（设计中有，实现中为降级模式）
