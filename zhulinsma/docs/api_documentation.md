# 竹林司马API接口文档

## 概述

竹林司马API是为杨总定制的专业级技术分析工具，提供全面的技术分析、数据质量监控和投资决策支持功能。系统采用模块化架构，支持双重验证机制，符合中国A股市场习惯（红涨绿跌）。

## 版本信息

- **版本**: 2.0.0
- **位置**: 广州
- **特点**: 双重验证机制、模块化架构、RESTful API、实时数据质量监控
- **核心原则**: 严谨性第一、数据真实性、双重审核、风险控制

## API模块架构

```
zhulinsma/
```

## API接口详解

### 1. 技术分析API (TechnicalAPI)

#### 核心功能

| 函数 | 描述 | 参数 | 返回值 |
|------|------|------|--------|
| `计算SMA` | 计算简单移动平均线 | 价格数据, 周期(默认20) | SMA值, 状态 |
| `计算EMA` | 计算指数移动平均线 | 价格数据, 周期(默认12) | EMA值, 状态 |
| `计算RSI` | 计算相对强弱指数 | 价格数据, 周期(默认14) | RSI值, 状态, 超买超卖信号 |
| `计算MACD` | 计算MACD指标 | 价格数据, 快线周期(12), 慢线周期(26), 信号周期(9) | MACD值, 信号线, 柱状图, 金叉死叉信号 |
| `计算布林带` | 计算布林带指标 | 价格数据, 周期(20), 标准差(2) | 上轨, 中轨, 下轨, 带宽, 位置 |
| `批量计算技术指标` | 批量计算多个技术指标 | 价格数据, 指标配置列表 | 所有指标结果 |

#### 使用示例

```python
from zhulinsma.interface.technical_api import TechnicalAPI

技术API = TechnicalAPI(验证模式=True)
价格数据 = [10.5, 10.8, 11.2, 10.9, 11.5, 11.8, 12.1, 11.9, 12.3, 12.5]

# 计算SMA
sma结果 = 技术API.计算SMA(价格数据, 周期=20)
if sma结果["成功"]:
    print(f"SMA(20) = {sma结果['数据']['值']}")

# 计算RSI
rsi结果 = 技术API.计算RSI(价格数据, 周期=14)
if rsi结果["成功"]:
    rsi值 = rsi结果['数据']['值']
    if rsi值 > 70:
        print(f"RSI({rsi值:.2f}) > 70，超买信号")
    elif rsi值 < 30:
        print(f"RSI({rsi值:.2f}) < 30，超卖信号")
```

### 2. 数据质量API (DataQualityAPI)

#### 核心功能

| 函数 | 描述 | 参数 | 返回值 |
|------|------|------|--------|
| `验证股票数据` | 验证单只股票数据质量 | 股票代码, 价格数据, 时间戳 | 质量评分, 检查结果, 建议 |
| `批量验证股票数据` | 批量验证多只股票数据 | 股票数据列表 | 每只股票的验证结果 |
| `获取监控状态` | 获取数据质量监控状态 | 无 | 监控指标, 报警状态 |
| `获取仪表板数据` | 获取监控仪表板数据 | 无 | 仪表板数据, 图表数据 |

#### 数据质量检查维度

1. **完整性检查**: 数据是否完整，有无缺失值
2. **有效性检查**: 数据是否有效，有无异常值（0、负数、极端值）
3. **一致性检查**: 数据是否一致，价格变化是否合理
4. **时效性检查**: 数据是否及时，时间戳是否连续
5. **分布检查**: 数据分布是否合理，有无异常模式

#### 使用示例

```python
from zhulinsma.interface.data_quality_api import DataQualityAPI

数据质量API = DataQualityAPI(验证模式=True)

# 验证数据质量
验证结果 = 数据质量API.验证股票数据(
    股票代码="000001.SZ",
    价格数据=[10.5, 10.8, 11.2, 10.9, 11.5, 11.8, 12.1],
    时间戳=["2026-03-01", "2026-03-02", "2026-03-03", "2026-03-04", 
           "2026-03-05", "2026-03-06", "2026-03-07"]
)

if 验证结果["成功"]:
    print(f"数据质量评分: {验证结果['质量评分']}/100")
    print(f"数据状态: {验证结果['状态']}")
    
    for 检查结果 in 验证结果["检查结果"]:
        if 检查结果["状态"] == "失败":
            print(f"⚠️ {检查结果['检查项']}: {检查结果['详情']}")
```

### 3. 综合分析API (AnalysisAPI)

#### 核心功能

| 函数 | 描述 | 参数 | 返回值 |
|------|------|------|--------|
| `执行技术分析` | 执行全面的技术分析 | 股票代码, 价格数据, 时间戳, 参数设置 | 数据质量验证 + 技术指标计算 + 趋势分析 + 风险评估 + 投资建议 |
| `批量技术分析` | 批量分析多只股票 | 股票数据列表 | 每只股票的分析结果 |
| `趋势分析` | 分析价格趋势 | 价格数据, 时间戳 | 趋势方向, 趋势强度, 支撑位, 阻力位 |
| `信号分析` | 分析技术信号 | 价格数据, 技术指标结果 | 买入信号, 卖出信号, 信号强度 |
| `风险评估` | 评估投资风险 | 价格数据, 技术指标结果, 市场环境 | 风险等级, 风险分数, 风险描述 |

#### 分析流程

1. **数据质量验证**: 检查数据完整性和有效性
2. **技术指标计算**: 计算SMA、EMA、RSI、MACD、布林带等指标
3. **趋势分析**: 识别上升/下降趋势，计算支撑阻力位
4. **信号分析**: 识别技术买卖信号
5. **风险评估**: 评估投资风险等级
6. **投资建议**: 生成操作建议

#### 使用示例

```python
from zhulinsma.interface.analysis_api import AnalysisAPI

分析API = AnalysisAPI(验证模式=True)

# 执行全面技术分析
分析结果 = 分析API.执行技术分析(
    股票代码="000001.SZ",
    价格数据=[10.5, 10.8, 11.2, 10.9, 11.5, 11.8, 12.1, 11.9, 12.3, 12.5],
    参数设置={
        "SMA周期": [5, 10, 20, 30, 60],
        "RSI周期": 14,
        "MACD参数": {"快线周期": 12, "慢线周期": 26, "信号周期": 9}
    }
)

if 分析结果["成功"]:
    print(f"📊 趋势分析: {分析结果['趋势分析']['趋势方向']}趋势")
    print(f"⚠️ 风险评估: {分析结果['风险评估']['风险等级']}风险")
    print(f"💡 投资建议: {分析结果['投资建议']['操作方向']}")
    
    # 显示详细技术指标
    for 指标名称, 指标数据 in 分析结果["技术指标结果"].items():
        if 指标数据 and 指标数据.get("成功", False):
            print(f"   {指标名称}: {指标数据['数据'].get('值', 'N/A')}")
```

### 4. Web RESTful API (WebAPI)

#### API端点

| 端点 | 方法 | 描述 | 请求参数 | 响应格式 |
|------|------|------|----------|----------|
| `/api/health` | GET | 健康检查 | 无 | JSON |
| `/api/analyze` | POST | 技术分析 | 股票代码, 价格数据, 时间戳 | JSON |
| `/api/batch-analyze` | POST | 批量分析 | 股票数据列表 | JSON |
| `/api/indicators/sma` | POST | 计算SMA | 价格数据, 周期 | JSON |
| `/api/indicators/rsi` | POST | 计算RSI | 价格数据, 周期 | JSON |
| `/api/indicators/macd` | POST | 计算MACD | 价格数据, 快线周期, 慢线周期, 信号周期 | JSON |

| `/api/system/status` | GET | 获取系统状态 | 无 | JSON |

## 错误处理

### 常见错误代码

| 状态码 | 错误类型 | 描述 | 解决方案 |
|--------|----------|------|----------|
| 400 | 参数错误 | 请求参数缺失或格式错误 | 检查参数格式，确保参数完整 |
| 401 | 验证失败 | 数据验证失败 | 检查数据质量，修复异常数据 |
| 403 | 权限不足 | API调用权限不足 | 检查API密钥或权限设置 |
| 404 | 资源不存在 | 请求的端点不存在 | 检查API端点URL |
| 500 | 服务器错误 | 服务器内部错误 | 联系系统管理员 |

### 错误处理示例

```python
from zhulinsma.external_api import 分析股票

try:
    结果 = 分析股票("000001.SZ", [10.5, 10.8, 11.2])
    
    if not 结果["成功"]:
        print(f"❌ 分析失败: {结果.get('消息', '未知错误')}")
        
        # 查看详细错误信息
        错误详情 = 结果.get("错误详情", {})
        if 错误详情:
            print(f"   错误类型: {错误详情.get('类型', '未知')}")
            print(f"   错误代码: {错误详情.get('代码', '未知')}")
            print(f"   建议: {错误详情.get('建议', '无')}")
        
        # 根据错误类型处理
        if "数据质量" in 结果.get("消息", ""):
            print("⚠️ 数据质量问题，建议:")
            print("   1. 检查数据完整性")
            print("   2. 移除异常值")
            print("   3. 确保时间戳连续")
    
except Exception as e:
    print(f"❌ 系统异常: {str(e)}")
    import traceback
    traceback.print_exc()
```

### 2. 错误处理

```python
from zhulinsma.external_api import 分析股票
import logging

# 配置日志
logging.basicConfig(level=logging.INFO)

def 安全分析股票(股票代码, 价格数据, 时间戳=None, 重试次数=3):
    """带重试机制的安全分析"""
    for 尝试 in range(重试次数):
        try:
            结果 = 分析股票(股票代码, 价格数据, 时间戳)
            
            if 结果["成功"]:
                return 结果
            else:
                logging.warning(f"第{尝试+1}次尝试失败: {结果.get('消息')}")
                
                # 如果是数据质量问题，尝试清理数据
                if "数据质量" in 结果.get("消息", ""):
                    价格数据 = 清理异常数据(价格数据)
                    
        except Exception as e:
            logging.error(f"第{尝试+1}次尝试异常: {str(e)}")
            
        if 尝试 < 重试次数 - 1:
            import time
            time.sleep(1)  # 等待1秒后重试
    
    return {"成功": False, "消息": f"经过{重试次数}次尝试后仍然失败"}

def 清理异常数据(价格数据):
    """清理价格数据中的异常值"""
    import numpy as np
    
    价格数组 = np.array(价格数据)
    
    # 计算统计信息
    均值 = np.mean(价格数组)
    标准差 = np.std(价格数组)
    
    # 定义异常值范围（均值±3倍标准差）
    下限 = 均值 - 3 * 标准差
    上限 = 均值 + 3 * 标准差
    
    # 过滤异常值
    过滤后数据 = [x for x in 价格数据 if 下限 <= x <= 上限]
    
    return 过滤后数据
```

## 常见问题解答

### Q1: 数据最少需要多少条？
**A**: 建议至少提供20条以上的价格数据以获得准确分析。最少需要5条数据，但分析结果可能不准确。

### Q2: 股票代码格式要求？
**A**: 必须包含交易所后缀：
- A股: `000001.SZ` (深圳), `600000.SH` (上海)
- 港股: `00700.HK`
- 美股: `AAPL.US`

### Q3: 如何处理异常值？
**A**: 系统会自动检测异常值，但建议在传入数据前进行预处理：
1. 移除0值、负值
2. 过滤极端值（如超过均值3倍标准差）
3. 确保价格变化合理

### Q4: 如何提高分析准确性？
**A**: 
1. 提供更多历史数据（至少50条）
2. 确保时间戳连续
3. 使用高质量数据源
4. 定期更新系统版本

### Q5: 如何集成到现有系统？
**A**: 
1. 使用Web API进行HTTP集成
2. 直接导入Python模块
3. 使用消息队列异步处理
4. 参考`examples/api_usage_example.py`
