## 一、现有系统分析

### 当前推荐引擎架构
- ✅ ItemCF (IUF + Cosine)
- ✅ UserCF (Cosine)
- ✅ Content-Based (TF-IDF + Tag)
- ✅ Trending Service
- ✅ Ranking Service（规则加权 + MMR）
- ✅ 缓存层（L1内存 + L2 Redis）
- ✅ SQLAlchemy ORM 模型
- ✅ FastAPI 完整 REST API

### 核心不足
| 问题 | 影响 |
|------|------|
| 排序依赖规则权重 | 无法自适应数据分布，效果天花板低 |
| 无Embedding协同 | 无法捕捉非线性协同信号 |
| 无特征工程管道 | 用户/物品动态特征缺失 |
| 无训练/评估体系 | 无法量化改进效果 |
| 无A/B测试 | 无法安全迭代 |

## 二、增强架构

```
┌─────────────────────────────────────────────────────┐
│                  用户请求 (FastAPI)                   │
└─────────────────────┬───────────────────────────────┘
                      │
         ┌────────────▼────────────┐
         │   多路召回 (Recall)       │
         │ ItemCF / UserCF / Content│
         │ Trending / Embedding CF  │
         └────────────┬────────────┘
                      │  候选集 (N × 特征)
         ┌────────────▼────────────┐
         │  特征工程管道 (Features)  │  ← 新增
         │  用户画像 / 物品特征     │
         │  交互序列 / 上下文特征   │
         └────────────┬────────────┘
                      │  特征向量
         ┌────────────▼────────────┐
         │   ML 排序模型 (Rank)    │  ← 核心新增
         │  LightGBM CTR 预测      │
         │  DeepFM / Wide&Deep     │
         └────────────┬────────────┘
                      │  排序得分
         ┌────────────▼────────────┐
         │   业务规则 + 多样性     │
         │   MMR + 曝光过滤        │
         └────────────┬────────────┘
                      │
         ┌────────────▼────────────┐
         │   A/B Test 分流         │  ← 新增
         │   指标监控 (CTR/时长)   │
         └─────────────────────────┘
```

## 三、模块设计

### 模块1：特征工程 (`rec/features/`)
- `user_profile_features.py` — 用户画像特征（偏好/活跃度/价值分）
- `item_features.py` — 物品统计特征（热度/质量/新鲜度）
- `context_features.py` — 上下文特征（时间/设备/场景）
- `sequence_features.py` — 序列特征（用户行为序列建模）

### 模块2：ML排序模型 (`rec/models/`)
- `ctr_model.py` — LightGBM点击率预测（主模型）
- `embedding_model.py` — Neural CF用户/物品Embedding

### 模块3：模型训练 (`rec/training/`)
- `feature_pipeline.py` — 数据 → 特征 → 训练集
- `trainer.py` — 模型训练 + 评估 + 版本管理
- `version_manager.py` — 模型版本控制

### 模块4：在线服务 (`rec/serving/`)
- `feature_store.py` — 实时特征服务
- `model_serving.py` — 模型推理服务
- `online_inference.py` — 在线推理API

### 模块5：A/B测试 (`rec/ab_test/`)
- `splitter.py` — 用户分流算法
- `metrics_collector.py` — 指标收集
- `experiment_manager.py` — 实验管理

## 四、技术指标

### 特征维度（预估）
| 类别 | 特征数 | 来源 |
|------|--------|------|
| 用户偏好特征 | ~30维 | 历史行为统计 |
| 物品统计特征 | ~25维 | 全局统计 |
| 上下文特征 | ~10维 | 请求上下文 |
| 序列特征 | ~50维 | 行为序列 |
| 协同特征 | ~20维 | Embedding |
| **合计** | **~135维** | |

### 模型选型
- **主模型**：LightGBM（可解释性强，训练快，适合表格特征）
- **备选**：XGBoost / CatBoost（可切换）
- **未来**：DeepFM（处理高维稀疏特征）

### 评估指标
- 离线：AUC、LogLoss、NDCG@K
- 在线：CTR、观看时长、留存率
